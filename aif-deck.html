AIF  ·  Adaptive Interface Framework

Interfaces are built for one kind of user.

— Simplified screens serve newcomers but frustrate power users who need depth and density.
— Complex screens serve specialists but overwhelm everyone else — and resist change.
— Most design systems solve this by building more screens. More variants. More maintenance.

There is a different approach. One that doesn't multiply screens — it makes each screen capable of being the right one.

02  ·  The Idea

One axis.
Everything scales on it.

Every screen, every container, every component sits somewhere on a single spectrum — from Minimal to Maximal. Context determines where.


Not different screens for different users. The same screen, reading the moment.

03  ·  The Structure

Three layers.
One shared axis.
Different degrees of freedom.

The Minimal→Maximal axis governs all three layers — but how rigidly each layer applies it differs. This is the system's flexibility and its discipline in the same idea.

04  ·  Layer 1 — Screen Structure
Entitlement-responsive

The screen layer scales in two directions.

Horizontal — how many hubs a user sees. Controlled by entitlements. A restricted user may have a single workspace: one function, no overview. A power user sees the full four-hub breadth.


Vertical — how much surfaces within any screen. A dashboard is not a fixed thing. For one user it's an overview of four domains. For another it's their entire world — a dedicated workspace for payment authorisation.


The word "dashboard" describes a role, not a screen. The framework holds both without building two products.

05  ·  Layer 2 — Container Sequence
Semi-rigid Position defines purpose

Where a container sits tells you what it does.

The sequence is largely fixed — but the mode of each container is not. A component in the bento can run at any mode. What the hierarchy enforces is the role of each slot, not the depth of what lives inside it.


— Strip — always monitoring. Always Minimal.
— Bento — signals and actions. Mixed modes.
— Grid — operational. Maximal. Optional.

06  ·  Layer 3 — Component Modes
Fluid Context decides the mode

Same component. Three states of depth.

Any component can run at any mode. A component in a monitoring dashboard defaults to Minimal. The same component in an operational workspace runs at Maximal. The component doesn't change — its context does.

Deliberately scoped.

The architecture is already capable of more. The scope is a product decision, not a design constraint.

Appendix A  ·  Extended Context

Entitlements & role-aware composition

Extended Context activates when the interface knows who it's talking to. Entitlements data — already in development — controls two distinct dimensions of the screen layer simultaneously.


This is not a new product. It's the same framework with richer inputs, unlocking what was always architecturally possible.

Horizontal — breadth of IA
— Entitlements control how many hubs a user sees
— A restricted user sees one function — their workspace is their dashboard
— A full user sees all four hubs from a true overview
Vertical — depth within screens
— Components unlock to Maximal mode for entitled users
— Operational Grid container activates when role warrants it
— Signals strip surfaces only what this user's scope can act on
Status
Entitlements system in active development. AIF architecture ready to receive on both axes.
Appendix B  ·  Frontier Technologies

The interface assembles itself.

At the frontier, the screen is no longer a designer's decision. Context — role, behaviour, signals, moment — assembles the right components in the right modes. The designer sets the grammar. The system speaks it.


This is the logical conclusion of the AIF framework: a fully exception-based interface that surfaces only what matters, composed on demand.

Capabilities it points toward
— Automated screen composition from context signals
— Agentic interface behaviour — takes action, surfaces results
— Exception-based surfacing — silence unless something matters
— Real-time mode adaptation — the component reads the moment
Status
Speculative — but architecturally grounded. The framework is built to get here.
